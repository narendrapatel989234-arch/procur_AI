ProcurAI — User Journey Map: Procurement Manager

Journey B  ·  Governance & oversight persona  ·  9 experience layers  ·  5 stages
Experience Layer Stage 1Entry & Oversight Stage 2Approval Review Stage 3Decision & Action Stage 4Portfolio Monitoring Stage 5Compliance & Governance
1. User Goal Get an immediate, accurate picture of all active procurements and pending decisions across the organization Fully understand the context and risk of a request before making a decision Make a fast, confident, and fully documented decision Maintain real-time visibility across all active lifecycles and spot bottlenecks before SLA breach Ensure all approved procurements are policy-compliant, auditable, and defensible
2. User Actions UserLog in → scan org-wide KPI cards → review pending approvals count → check SLA at-risk items → navigate to Approvals tab UserOpen approval item → read AI summary → review full PR payload → check compliance snapshot → preview linked documents if needed UserSelect decision (Approve / Reject / Request Changes / Escalate) → add structured feedback if returning → confirm decision UserReview active requests table (org-wide) → filter by SLA at-risk → check stage distribution → drill into delayed PRs UserReview compliance validation results on SoW tab → access Activity & Audit tab → export audit trails when needed
3. System Interactions AI AutoRole-filtered Dashboard loads with org-wide KPIs → pending approval queue pre-sorted by SLA risk + value → SLA breach alerts surfaced automatically AI AutoApproval detail loads with full payload pre-populated → AI generates request summary with source citations → compliance status snapshot surfaced inline → budget validation status shown automatically AI AutoDecision recorded with timestamp + actor immediately → workflow state updated automatically → structured feedback routed to analyst → audit log entry created without manual input AI AutoAggregated lifecycle view maintained in real time → SLA status per PR updated continuously → stage distribution surfaced without user input AI AutoCompliance engine runs and stores results per SoW automatically → full audit trail maintained with timestamps → approval history logged with actor, reason, and timestamp
4. Touchpoints Login · Dashboard (manager KPIs) · Active Requests table · Approvals tab · SLA indicators · Pending approvals count Approvals tab · Approval detail view · AI summary panel · Compliance snapshot · Budget validation · Linked document preview Approval detail — decision action buttons · Reject reason field · Request changes structured form · Escalation routing · Activity & Audit tab Dashboard (manager view) · Active Requests table · SLA status column · Lifecycle stage column · Filters · PR Detail Page (drill-in) PR Detail Page (SoW tab — Compliance panel) · Activity & Audit tab · Approval History · Export audit trail action
5. User Thoughts "How many approvals are waiting? What is urgent? What is highest value? Is anything at SLA risk today?" "Is this complete and compliant? What are the risks? Do I need to open the full PR or is the summary enough?" "Am I making the right call? Is my reasoning documented? Will the analyst know exactly what to fix if I return this?" "Is anything falling behind? Where are the bottlenecks? Are there patterns in what is getting stuck across the team?" "Is this procurement policy-compliant? Can I defend every decision if this is audited? Is the audit trail complete?"
6. User Emotions Authoritative, time-pressured Analytical, cautious Decisive, occasionally uncertain on complex items Strategic, mildly concerned about portfolio health Responsible, risk-averse
7. Emotion Intensity Medium Medium–High Medium Low–Medium Medium–High
8. Pain Points
  • No priority ranking in approval queue
  • Unable to distinguish urgent from routine at a glance
  • KPI cards too generic without drill-down
  • Missing context requiring manual PR navigation
  • AI summaries without citations reduce trust
  • Compliance status not immediately visible
  • Free-text rejection with no structure
  • No modify-before-approve option
  • Audit trail not immediately visible after decision
  • No aggregated bottleneck view across all PRs
  • Cycle time data requires opening individual PRs
  • No way to see patterns across the portfolio
  • Compliance findings buried in SoW view
  • Audit trail requires navigating each PR separately
  • No consolidated compliance report
9. Opportunities
  • AI-prioritized approval queue (SLA + value weighted)
  • One-line AI summary per pending item
  • Quick-approve for low-risk routine items from queue
  • Cited AI summary in approval view
  • Visual compliance status block (pass/fail per checklist)
  • Document preview without leaving panel
  • Structured feedback form for returned items
  • Modify-before-approve capability
  • Instant audit confirmation after decision
  • Suggested next approver on escalation
  • SLA at-risk filter as default saved view
  • Bottleneck pattern flag in KPIs
  • Stage distribution summary above table
  • Compliance status badge on PR list rows
  • One-click exportable audit trail
  • Compliance summary in approval view before decision
